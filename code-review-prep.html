Background

We are DJ Phat Enchantah. Our goal is to create a DJ that will take your music and create the swaggiest, most hip mix of your songs.

Echonest: We recently discovered Echonest following the design review. It contains a multitude of libraries for music developers across different platforms. Moreover, its API holds an array of data consisting of a song’s components such as its beat, pitch, key, etc. Echonest Remix: What Echonest offers that does some incredible things is its Echonest Remix API. This allows someone to “remix, re-edit, and reimagine any piece of music and video.” Some examples people have created using remix:

  • play a song forever
  • reversing everything in a song
  • beat matching

What we plan to talk about:

  • We have created a macro-layout of our code, separated into three sections: API (data retrieval) → we will show you this during the review
  • Analyzing songs (for doing cool things with it)**
  • Main code (for putting everything together)

Main things to think about:

  • storing the data we retrieve (We have thought of something but we want your unbiased opinion on it, so please think about it. Lists? Dictionaries? Tuples? A combination?)
  • analysis of the data** (best way to find the chorus? Default time for song snippet? Knowing when to transition? What transitions to use: fade-ins?)
